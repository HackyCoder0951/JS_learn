# JavaScript Execution Context (Practical Point of View)

## What is Execution Context?

Execution Context is the **environment where JavaScript code is executed**.

Whenever JavaScript runs code, it creates a special container called an **Execution Context**.

This container stores:

| Component | Purpose |
|---|---|
| **Memory Component** | Stores variables and functions |
| **Code Component** | Executes code line by line |

---

# Practical Real-Life Analogy

Imagine JavaScript like a **chef in a kitchen**.

---

## Memory Component = Storage Shelf

The chef stores ingredients before cooking.

```js
let a = 10;

function greet() {
    console.log("Hello");
}
```

Memory becomes:

| Key | Value |
|---|---|
| a | 10 |
| greet | function |

---

## Code Component = Chef Cooking Step-by-Step

Now chef executes instructions one by one.

```js
console.log(a);
greet();
```

Execution flow:

```txt
Step 1 -> console.log(a)
Step 2 -> greet()
```

---

# Important Concept

JavaScript does **NOT** execute all code at once.

It executes:

```txt
Line by line
Top to bottom
One at a time
```

That is why JS is called:

# JavaScript is Synchronous

Meaning:

> One task finishes first, then next task starts.

---

# JavaScript is Single-Threaded

Meaning:

> JS has only ONE call stack / ONE worker.

Only one thing can execute at a time.

---

# Example 1 — Understanding Execution Context

```js
let a = 10;
let b = 20;

function add(x, y) {
    return x + y;
}

let result = add(a, b);

console.log(result);
```

---

# Phase 1 → Memory Creation Phase

Before execution starts:

Memory stores:

| Variable | Value |
|---|---|
| a | undefined |
| b | undefined |
| add | function |
| result | undefined |

---

# Phase 2 → Code Execution Phase

Now JS executes line-by-line.

---

## Step 1

```js
let a = 10;
```

Memory:

```js
a = 10
```

---

## Step 2

```js
let b = 20;
```

Memory:

```js
b = 20
```

---

## Step 3

Function already stored in memory.

---

## Step 4

```js
let result = add(a, b);
```

Function executes.

New Function Execution Context is created.

---

# Inside Function Execution Context

```js
x = 10
y = 20
```

Returns:

```js
30
```

Then function context is removed from stack.

---

## Final Output

```js
30
```

---

# Visual Flow

```txt
Global Execution Context Created

Memory:
a -> 10
b -> 20
add -> fn()

Code Execution:
↓
add(10,20)
↓
Function Execution Context
↓
return 30
↓
console.log(30)
```

# Example 2 — Why JS is Single Threaded

```js
console.log("Start");

for(let i = 0; i < 1000000000; i++) {
    // heavy task
}

console.log("End");
```

---

# What Happens?

JS executes:

```txt
1. Start
2. Huge loop
3. End
```

Browser freezes until loop completes.

Why?

Because JS can only do ONE task at a time.

---

# Output

```txt
Start
(wait...)
End
```

# Example 3 — Understanding Synchronous Nature


Output:

Always in order.

# Real Interview Understanding

---

# Why Does JS Need Execution Context?

Because JS needs:

- Memory for variables/functions
- A place to execute code
- A way to track function calls

---

# Types of Execution Context

| Type | Description |
|---|---|
| Global Execution Context | Created when JS file starts |
| Function Execution Context | Created whenever function runs |
| Eval Execution Context | Created by `eval()` |

---

# Important Internal Steps

Whenever execution context is created:

## 1. Memory Allocation

Variables stored as:

```js
undefined
```

Functions stored fully in memory.

---

## 2. Code Execution

Code runs line-by-line.

Values assigned.

Functions invoked.

---

# Example of Hoisting

```js
console.log(a);

var a = 10;
```

Output:

```txt
undefined
```

Why?

During memory creation:

```js
a = undefined
```

Then execution phase assigns:

```js
a = 10
```

# Another Practical Example

```js
var username = "Hacky";

function showUser() {
    var age = 25;

    console.log(username);
    console.log(age);
}

showUser();
```

---

# Global Execution Context

Memory:

```js
username -> undefined
showUser -> function
```

Execution:

```js
username = "Hacky"
showUser()
```

---

# Function Execution Context

Memory:

```js
age -> undefined
```

Execution:

```js
age = 25
console.log(username)
console.log(age)
```

---

# Output

```txt
Hacky
25
```

# Core Understanding Summary

| Concept | Meaning |
|---|---|
| Execution Context | Environment where code executes |
| Memory Component | Stores variables/functions |
| Code Component | Executes code |
| Synchronous | Executes one-by-one |
| Single Threaded | Only one task at a time |
| Call Stack | Tracks function execution |
| Global Context | Default execution environment |
| Function Context | Created per function call |

# Golden Rule

```txt
JavaScript first stores everything in memory,
then executes code line-by-line.
```
