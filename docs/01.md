# JavaScript Variables (`var`, `let`, `const`)

---

# 01 - Variables

A variable is a **named storage** for data.  
We can use variables to store values like user names, numbers, objects, arrays, etc.

## Types of Variable Declarations

### 1. `var`

```js
var variableName = value;
```

- Can be reassigned
- Function scoped
- Old way of declaring variables

Example:

```js
var city = "Jaipur";
city = "Delhi";

console.log(city);
```

---

### 2. `const`

```js
const variableName = value;
```

- Cannot be reassigned
- Block scoped
- Must be initialized during declaration

Example:

```js
const country = "India";

console.log(country);
```

---

### 3. `let`

```js
let variableName = value;
```

- Can be reassigned
- Block scoped
- Modern variable declaration

Example:

```js
let age = 25;
age = 30;

console.log(age);
```

---

> If variables are not created inside a function or block, they become globally scoped.

---

# 02 - JS `let`

---

## 1. Block Scope

Variables declared inside a block `{}` cannot be accessed outside the block.

```js
{
    let name = "Jignesh";
    console.log(name);
}

console.log(name); // Error
```

---

## 2. Global Scope

Variables declared with `var` have global scope.

```js
{
    var city = "Jaipur";
}

console.log(city);
```

Output:

```js
Jaipur
```

`var` does NOT support true block scope.

---

## 3. Cannot Be Redeclared

Variables declared using `let` cannot be redeclared in the same scope.

❌ Invalid:

```js
let x = "John Doe";
let x = 0;
```

✅ Valid with `var`:

```js
var x = "John Doe";
var x = 0;
```

---

## 4. Redeclaring Variables

Redeclaring variables with `var` can create issues.

### Problem with `var`

```js
var language = "JavaScript";

{
    var language = "Python";
}

console.log(language);
```

Output:

```js
Python
```

The outer variable also changes.

---

### Solution with `let`

```js
let language = "JavaScript";

{
    let language = "Python";
}

console.log(language);
```

Output:

```js
JavaScript
```

---

## 5. What is Good About `let` and `const`

- Block scoped
- Cannot be redeclared
- Must be declared before use
- Do not bind to `this`
- Safer for modern JavaScript

---

## 6. Problems with `var`

- No block scope
- Hoisted
- Can create unexpected bugs
- Binds to `this`

---

## 7. Redeclaring Rules

### `var` Redeclaration Allowed

```js
var user = "Jignesh";
var user = "Amit";
```

---

### `let` Redeclaration NOT Allowed

```js
let user = "Jignesh";
let user = "Amit"; // Error
```

---

### Redeclaration in Different Blocks Allowed

```js
let user = "Jignesh";

{
    let user = "Amit";
    console.log(user);
}

console.log(user);
```

Output:

```js
Amit
Jignesh
```

# 03 - JS `const`

---

## 1. Constant Objects and Arrays

`const` defines a constant reference, not a completely immutable value.

### You CANNOT:

- Reassign a constant value
- Reassign a constant array
- Reassign a constant object

### You CAN:

- Modify array elements
- Modify object properties

---

### Constant Array Example

```js
const fruits = ["Apple", "Banana"];

fruits.push("Mango");

console.log(fruits);
```

Output:

```js
["Apple", "Banana", "Mango"]
```

---

### Constant Object Example

```js
const user = {
    name: "Jignesh"
};

user.name = "Amit";

console.log(user);
```

Output:

```js
{ name: "Amit" }
```

---

## 2. What is Good About `const`

- Block scoped
- Cannot be redeclared
- Must be declared before use
- Safer for fixed values

---

## 3. Problems with `var`

- Does not require declaration
- Hoisted
- Function scoped

---

## 4. Block Scope

`const` behaves similar to `let` regarding block scope.

```js
{
    const city = "Delhi";
}

console.log(city); // Error
```

---

## 5. Redeclaring Rules

### Allowed with `var`

```js
var x = 10;
var x = 20;
```

---

### NOT Allowed

```js
let x = 10;
const x = 20; // Error
```

---

### Reassignment NOT Allowed

```js
const PI = 3.14;

PI = 5; // Error
```

---

### Allowed in Different Scope

```js
const name = "Jignesh";

{
    const name = "Amit";
    console.log(name);
}

console.log(name);
```

---

## 6. Hoisting

### `var` Hoisting

```js
console.log(a);

var a = 10;
```

Output:

```js
undefined
```

---

### `const` Hoisting

```js
console.log(a);

const a = 10;
```

Output:

```js
ReferenceError
```

---

## 7. Uppercase Constants

Constants known before execution are generally written in uppercase.

Example:

```js
const COLOR_RED = "#F00";
const COLOR_GREEN = "#0F0";
const COLOR_BLUE = "#00F";
const COLOR_ORANGE = "#FF7F00";
```

---

## Example Usage

```js
let color = COLOR_ORANGE;

alert(color);
```

Output:

```js
#FF7F00
```

---

## Benefits

- Easier to remember
- More readable
- Prevents magic values
- Improves maintainability

---

## Runtime Constant Example

```js
const pageLoadTime = "Time taken by webpage to load";
```

This value is determined during execution but does not change afterward.

# Variable Naming Best Practices

Variable names should clearly describe the stored data.

---

## Good Naming Rules

### 1. Use Human Readable Names

```js
let userName;
let shoppingCart;
```

---

### 2. Avoid Unnecessary Short Names

❌ Bad:

```js
let a;
let b;
```

✅ Good:

```js
let userAge;
let totalPrice;
```

---

### 3. Use Descriptive Names

❌ Bad:

```js
let data;
let value;
```

✅ Good:

```js
let employeeSalary;
let customerAddress;
```

---

### 4. Maintain Naming Consistency

If users are called `user`, continue using:

```js
currentUser
newUser
```

instead of:

```js
currentVisitor
newManInTown
```

---

# JavaScript Naming Conventions

---

## 1. Pascal Case

Every word starts with uppercase.

```js
SmartWatch
MobileDevice
WebDriver
```

Used mainly for:

- Classes
- Constructors

---

## 2. Camel Case (Most Common)

First word lowercase, next words uppercase.

```js
smartWatch
mobileDevice
webDriver
```

Used for:

- Variables
- Functions

---

## 3. Snake Case / Underscore

Words separated using `_`.

```js
smart_watch
mobile_device
web_driver
```

# Why `var` is Not Preferred

```js
if (true) {
    var test = "Hello";
}

console.log(test);
```

Output:

```js
Hello
```

This creates scope issues because `var` ignores block scope.

---

# Modern JavaScript Recommendation

✅ Use `const` by default  
✅ Use `let` when value changes  
❌ Avoid using `var`

---

# Summary

| Feature | var | let | const |
|---|---|---|---|
| Scope | Function Scoped | Block Scoped | Block Scoped |
| Reassign Allowed | Yes | Yes | No |
| Redeclare Allowed | Yes | No | No |
| Hoisted | Yes | Yes | Yes |
| Use Before Declaration | undefined | Error | Error |
| Preferred in Modern JS | No | Yes | Yes |

---

# Final Notes

- `let` is used for changing values
- `const` is used for fixed values
- `var` is old and generally avoided
- Good variable names improve code readability and maintainability
