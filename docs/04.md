# JavaScript Comparisons and Equality Operators

---

# Introduction

JavaScript provides comparison operators to compare values.

Comparison operators return:

- `true`
- `false`

depending on the comparison result.

---

# Basic Comparison Operators

| Operator | Meaning |
|---|---|
| `>` | Greater than |
| `<` | Less than |
| `>=` | Greater than or equal |
| `<=` | Less than or equal |
| `==` | Loose equality |
| `!=` | Not equal |
| `===` | Strict equality |

---

# Basic Examples

```js
console.log(2 > 1);
console.log(2 >= 1);
console.log(2 < 1);
console.log(2 == 1);
console.log(2 != 1);
console.log(2 <= 1);
```

---

# Output

```js
true
true
false
false
true
false
```

---

# String and Number Comparison

JavaScript automatically converts strings into numbers during comparisons.

---

# Example

```js
console.log("2" > 1);
```

Output:

```js
true
```

Explanation:

```js
"2" → 2
```

So:

```js
2 > 1
```

becomes:

```js
true
```

---

# Example with Leading Zero

```js
console.log("02" > 1);
```

Output:

```js
true
```

Explanation:

```js
"02" → 2
```

Then:

```js
2 > 1
```

---

# Comparison with `null`

---

# Example

```js
console.log(null > 0);
console.log(null == 0);
console.log(null >= 0);
```

---

# Output

```js
false
false
true
```

---

# Why Does This Happen?

JavaScript handles:

- equality (`==`)
- comparisons (`>`, `<`, `>=`, `<=`)

differently.

---

# Important Rule

During numeric comparison:

```js
null → 0
```

---

# Step-by-Step Explanation

---

## 1. `null > 0`

```js
null > 0
```

becomes:

```js
0 > 0
```

Output:

```js
false
```

---

## 2. `null == 0`

Equality behaves differently.

```js
null == 0
```

Output:

```js
false
```

Because:

- `null` is only loosely equal to:
  - `undefined`
  - itself

---

## 3. `null >= 0`

Comparison converts:

```js
null → 0
```

So:

```js
0 >= 0
```

Output:

```js
true
```

---

# Comparison with `undefined`

---

# Example

```js
console.log(undefined == 0);
console.log(undefined == undefined);
console.log(undefined > 0);
console.log(undefined < 0);
```

---

# Output

```js
false
true
false
false
```

---

# Explanation

---

## 1. `undefined == 0`

```js
false
```

`undefined` does not equal `0`.

---

## 2. `undefined == undefined`

```js
true
```

JavaScript considers undefined values equal to each other.

---

## 3. `undefined > 0`

```js
false
```

---

## 4. `undefined < 0`

```js
false
```

---

# Why?

Numeric conversion of `undefined` becomes:

```js
NaN
```

And any comparison involving `NaN` returns:

```js
false
```

---

# Loose Equality (`==`)

The `==` operator compares values after type conversion.

---

# Example

```js
console.log("2" == 2);
```

Output:

```js
true
```

Because:

```js
"2" → 2
```

Then:

```js
2 == 2
```

---

# Strict Equality (`===`)

The `===` operator checks:

1. Value
2. Data type

No automatic type conversion happens.

---

# Example

```js
console.log("2" === 2);
```

Output:

```js
false
```

Because:

| Value | Type |
|---|---|
| `"2"` | string |
| `2` | number |

Different types → result is `false`.

---

# Difference Between `==` and `===`

| Operator | Type Conversion | Checks Type |
|---|---|---|
| `==` | Yes | No |
| `===` | No | Yes |

---

# Example Comparison Table

| Expression | Result |
|---|---|
| `"2" == 2` | `true` |
| `"2" === 2` | `false` |
| `null == undefined` | `true` |
| `null === undefined` | `false` |

---

# Best Practice

✅ Prefer using `===` and `!==`  
❌ Avoid `==` because automatic type conversion can cause unexpected results

---

# Recommended Usage

```js
if (userAge === 18) {
    console.log("Adult");
}
```

---

# Important Notes

---

## `null`

- Converts to `0` in numeric comparisons
- Not equal to `0`

---

## `undefined`

- Converts to `NaN`
- Comparisons return `false`

---

## `===`

- Safer
- Predictable
- Preferred in modern JavaScript

---

# Summary

JavaScript comparison operators behave differently depending on:

- data types
- equality type
- automatic conversion rules

---

# Key Concepts

| Concept | Description |
|---|---|
| `==` | Loose equality with conversion |
| `===` | Strict equality without conversion |
| `null` | Converts to `0` in comparisons |
| `undefined` | Converts to `NaN` |
| `NaN` comparisons | Always `false` |

---

# Final Recommendation

✅ Use:

```js
===
```

instead of:

```js
==
```

because strict equality avoids unexpected bugs caused by automatic type conversion.
