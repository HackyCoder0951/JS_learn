# Chapter 2 - Value vs Reference in JavaScript

---

# Introduction

## What is Value vs Reference?

- JavaScript stores data in two ways:
  1. By Value
  2. By Reference

Understanding this concept is very important because it affects:
- Variables
- Arrays
- Objects
- Functions
- Memory handling
- React state updates

---

# JavaScript Data Types

## Primitive Data Types

- Stored by **Value**

### Types

- String
- Number
- Boolean
- Null
- Undefined
- BigInt
- Symbol

---

## Non-Primitive Data Types

- Stored by **Reference**

### Types

- Object
- Array
- Function

---

# Primitive Types → Stored By Value

## Description

- A copy of the actual value is stored.
- Changing one variable does not affect another.

---

# Example

```js
{
    let a = 10;

    let b = a;

    b = 20;

    console.log(a);

    console.log(b);
}
```

---

# Memory Representation

```text
a = 10
b = 10

After change:

a = 10
b = 20
```

- Both variables are independent.

---

# Real World Example

```js
{
    let password = "admin123";

    let copiedPassword = password;

    copiedPassword = "newPassword";

    console.log(password);

    console.log(copiedPassword);
}
```

---

# Non-Primitive Types → Stored By Reference

## Description

- Variables store memory address (reference).
- Multiple variables can point to same object.

---

# Example with Object

```js
{
    let user1 = {
        name: "Jignesh"
    };

    let user2 = user1;

    user2.name = "Rahul";

    console.log(user1);

    console.log(user2);
}
```

---

# Memory Representation

```text
user1 ------
             \
              { name: "Rahul" }
             /
user2 ------
```

- Both variables point to same object.

---

# Example with Array

```js
{
    let arr1 = [1, 2, 3];

    let arr2 = arr1;

    arr2.push(4);

    console.log(arr1);

    console.log(arr2);
}
```

---

# Why This Happens?

- Arrays and objects are stored in heap memory.
- Variables store reference addresses.

---

# Stack vs Heap Memory

| Stack Memory | Heap Memory |
|---|---|
| Primitive values | Objects & arrays |
| Faster | Slower |
| Fixed size | Dynamic size |
| Stored directly | Stored by reference |

---

# Stack Memory Example

```js
{
    let a = 10;

    let b = a;
}
```

---

# Heap Memory Example

```js
{
    let user = {
        name: "Jignesh"
    };
}
```

---

# Copying Arrays Properly

## Problem

```js
{
    let arr1 = [1, 2, 3];

    let arr2 = arr1;

    arr2.push(4);

    console.log(arr1);
}
```

### Output

```js
[1, 2, 3, 4]
```

---

# Solution 1 - Using Spread Operator

## Example

```js
{
    let arr1 = [1, 2, 3];

    let arr2 = [...arr1];

    arr2.push(4);

    console.log(arr1);

    console.log(arr2);
}
```

---

# Solution 2 - Using `slice()`

## Example

```js
{
    let arr1 = [1, 2, 3];

    let arr2 = arr1.slice();

    arr2.push(4);

    console.log(arr1);

    console.log(arr2);
}
```

---

# Copying Objects Properly

## Problem

```js
{
    let user1 = {
        name: "Jignesh"
    };

    let user2 = user1;

    user2.name = "Rahul";

    console.log(user1);
}
```

---

# Solution - Spread Operator

## Example

```js
{
    let user1 = {
        name: "Jignesh"
    };

    let user2 = {
        ...user1
    };

    user2.name = "Rahul";

    console.log(user1);

    console.log(user2);
}
```

---

# Shallow Copy

## Description

- Copies only first level.
- Nested objects are still referenced.

---

# Example

```js
{
    let user1 = {
        name: "Jignesh",
        address: {
            city: "Delhi"
        }
    };

    let user2 = {
        ...user1
    };

    user2.address.city = "Mumbai";

    console.log(user1);

    console.log(user2);
}
```

---

# Why Problem Happens?

```text
Nested objects still share same reference.
```

---

# Deep Copy

## Description

- Creates completely independent copy.
- Nested objects are also copied.

---

# Deep Copy Using `structuredClone()`

## Example

```js
{
    let user1 = {
        name: "Jignesh",
        address: {
            city: "Delhi"
        }
    };

    let user2 =
        structuredClone(user1);

    user2.address.city = "Mumbai";

    console.log(user1);

    console.log(user2);
}
```

---

# Deep Copy Using JSON

## Example

```js
{
    let user1 = {
        name: "Jignesh",
        age: 21
    };

    let user2 = JSON.parse(
        JSON.stringify(user1)
    );

    user2.name = "Rahul";

    console.log(user1);

    console.log(user2);
}
```

---

# Difference Between Shallow Copy and Deep Copy

| Shallow Copy | Deep Copy |
|---|---|
| Copies first level only | Copies complete structure |
| Nested references shared | Fully independent |
| Faster | Slower |

---

# Function Argument Behavior

## Primitive Example

```js
{
    function changeValue(x) {

        x = 100;
    }

    let num = 10;

    changeValue(num);

    console.log(num);
}
```

### Output

```js
10
```

---

# Object Example

```js
{
    function changeUser(user) {

        user.name = "Rahul";
    }

    let user = {
        name: "Jignesh"
    };

    changeUser(user);

    console.log(user);
}
```

### Output

```js
{
    name: "Rahul"
}
```

---

# Real World Example - React State Problem

```js
{
    let state = {
        count: 0
    };

    let newState = state;

    newState.count++;

    console.log(state.count);
}
```

---

# Correct React Style

```js
{
    let state = {
        count: 0
    };

    let newState = {
        ...state,
        count: state.count + 1
    };

    console.log(state);

    console.log(newState);
}
```

# Important Interview Questions

---

# Q1 - Are Arrays Stored by Value or Reference?

## Answer

```text
Arrays are stored by reference.
```

---

# Q2 - Are Objects Mutable?

## Answer

```text
Yes, objects are mutable.
```

---

# Q3 - What is Difference Between Stack and Heap?

| Stack | Heap |
|---|---|
| Primitive values | Objects & arrays |
| Fast access | Dynamic memory |
| Stored directly | Stored by reference |

---

# Q4 - What is Shallow Copy?

## Answer

```text
Copying only first level while nested references remain shared.
```

---

# Q5 - What is Deep Copy?

## Answer

```text
Completely independent copy including nested objects.
```

---

# Important Notes

- Primitive values are immutable.
- Objects and arrays are mutable.
- Spread operator is commonly used for copying.
- Deep copy is important in:
  - React
  - Redux
  - APIs
  - State management
