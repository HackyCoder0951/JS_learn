# JavaScript Basics and Data Types

---

# `"use strict"` Mode

```js
"use strict";
```

The `"use strict"` directive enables **strict mode** in JavaScript.

Strict mode helps:

- Write cleaner code
- Catch common coding mistakes
- Prevent unsafe actions
- Use newer JavaScript standards

Example:

```js
"use strict";

x = 10; // Error because variable is not declared
```

Without strict mode, JavaScript may allow such mistakes.

---

# Alert Message Box

```js
alert("Hello");
```

- `alert()` displays a popup message box in browsers.
- It works only in browser environments.

---

## Node.js Note

Node.js does not support browser popup methods like `alert()`.

Instead, Node.js uses:

```js
console.log("Hello");
```

---

# Automatic Semicolon Insertion (ASI)

JavaScript automatically inserts semicolons in many cases.

Example:

```js
let age = 20
console.log(age)
```

JavaScript interprets it as:

```js
let age = 20;
console.log(age);
```

---

## Important Note About ASI

Automatic semicolon insertion does NOT always work correctly.

Example:

```js
console.log(30 +
    1
    + 35);
```

Output:

```js
66
```

JavaScript understands this as one complete expression because the `+` operator indicates continuation.

# Object

Example:


---

# Symbol

Symbols create unique identifiers.

Example:

```js
const id = Symbol("123");
const anotherId = Symbol("123");

console.log(id === anotherId);
```

Output:

```js
false
```

Even if descriptions are same, symbols remain unique.


```js

```

---

```js

```

Output:

```js

```

---

```js

```

Output:

```js
"string"
```

---

```js

```

Output:

```js
"symbol"
```

---

```js
console.log(typeof Math);
```

Output:

```js
"object"
```

---

```js

```

Output:

```js
"object"
```

> This is a historical bug in JavaScript.

---

```js

```

Output:

```js
"function"
```

Functions are technically objects, but `typeof` returns `"function"`.

# Stack vs Heap Memory

---

# Primitive Types → Stack Memory

Primitive values are copied.

Example:

```js
let myYoutubeName = "jigneshgoogle.com";

let anotherName = myYoutubeName;

anotherName = "jignesh";

console.log(myYoutubeName);
console.log(anotherName);
```

Output:

```js
jigneshgoogle.com
jignesh
```

Original value remains unchanged.

---

# Non-Primitive Types → Heap Memory

Objects share references.

Example:

```js
let userOne = {
    email: "jignesh@google.com",
    upi: "user@ybl"
};

let userTwo = userOne;

userTwo.email = "jignesh@ssci.com";

console.log(userOne.email);
console.log(userTwo.email);
```

Output:

```js
jignesh@ssci.com
jignesh@ssci.com
```

Both variables point to the same object reference.
